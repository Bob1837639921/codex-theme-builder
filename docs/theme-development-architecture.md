# 主题开发架构

这套架构的目标不是复刻某一种水墨或冰雪风格，而是让 Codex 能从文字、截图、设计稿或既有主题出发，自动完成可实现设计、开发、热预览、视觉修正、验证和打包。

## 分层与依赖方向

```text
桌面快捷方式
  -> desktop-launch.ps1         启动编排、确认与错误边界
     -> ui/launcher-ui.ps1      启动窗口与真实进度显示
     -> launch.ps1              Codex 启动、CDP 验证与注入生命周期
        -> injector.mjs         安全载入主题目录和共享运行时
           -> base.css          跨主题结构修复与稳定层级
           -> runtime.js        DOM 标记、主题切换和低成本状态同步
           -> themes/<id>/      视觉身份、主题资源和局部样式
```

依赖只向下流动。主题目录不能处理进程、端口或 CDP；启动界面不能探测 Codex DOM；共享运行时不能写死某套主题的颜色、名称或图片。

## 一个主题包包含什么

```text
assets/themes/<theme-id>/
├─ theme.json                  # 身份、颜色与本地资源清单
├─ theme.css                   # 只属于该主题的样式
├─ home.webp                   # 首页全画布构图
├─ conversation.webp           # 对话工作区构图
├─ sidebar.webp                # 可选侧栏回退纹理
├─ selected-marker.webp        # 可选当前任务标记
├─ composer-edge.webp          # 可选输入框前景装饰
├─ icon-build.svg
├─ icon-analyze.svg
├─ icon-automate.svg
└─ icon-debug.svg
```

所有文件都位于主题自身目录。新增主题时把 ID 加入相邻的 `theme-catalog.json`，运行时会自动发现，不需要修改主题数量或写新的切换器逻辑。

## 新主题必须覆盖的界面

| 界面 | 主题负责 | 共享运行时负责 |
|---|---|---|
| 全局画布 | 构图、底色、可读安全区 | 根类与首页/对话路由标记 |
| 侧栏 | 普通、悬停、焦点、选中、折叠控制配色 | 稳定标记当前任务和折叠控件 |
| 首页 | 全屏场景、标题、四个入口视觉 | 挂载全宽 overlay，保留原生项目选择器 |
| 对话 | 正文、代码、图片、工具结果的对比度 | 识别工作区和滚动状态 |
| 文件变更 | 摘要胶囊与背景融合 | 清理原生不透明渐变轨道 |
| 输入框 | 表面、边缘装饰、焦点与多行视觉 | 保留附件、模型、麦克风、提交/停止操作 |
| 当前任务 | 标记、边框和选中色 | 把标记绑定到标题并复用节点，避免闪烁 |
| 输出面板 | 外壳、标题、来源与缩略图表面 | 只在面板出现时标记目标容器 |
| 弹出层 | 菜单、用量卡、对话框和禁用文字对比度 | 保留 portal 交互与原生焦点流 |
| 主题切换 | 卡片视觉和主题身份 | 目录读取、持久化、键盘、失败回滚 |

连续场景主题只在全局画布挂载一次主图。侧栏、标题栏、正文和输出面板叠加透明表面，不能在多个容器中分别 `background-size: cover`，否则人物、建筑或乐器会被二次裁切。

## 自动化开发流程

1. 分析需求和 Codex 截图，锁定不能移动或重建的原生控件。
2. 将每个视觉元素映射为 DOM hook、运行时标记、主题资产或交互状态。
3. 没有明确视觉目标时生成三个可实现方向；用户授权自动决策时按适配、可读、成本和可维护性评分后继续。
4. 用 `scripts/new-theme.ps1` 创建自包含主题包，优先生成独立首页图和对话图。
5. 逐项实现上面的十类界面；共享能力放入 runtime，视觉身份留在主题目录。
6. 用 `scripts/test-theme.ps1` 做静态和 payload 校验。
7. 用 `scripts/preview-theme.ps1` 热应用并检查首页、长对话、窄窗口、文件变更、运行状态、输出面板与弹出层。
8. 修正所有高、中影响差异，记录 `final result: passed` 后才打包。
9. 运行仓库验证，安装 Skill，确认安装副本与仓库源文件一致。

## 资源与性能预算

- 首页、对话和侧栏优先使用质量受控的 WebP。
- 单张主题位图上限为 8 MB；实际应尽量控制在 500 KB 内，除非视觉比较证明必须更大。
- 选中标记不超过 512 KB，且尺寸应接近真实渲染大小。
- 输入框前景图不超过 2 MB，中间和原生控制区必须透明。
- SVG 图标不超过 64 KB，不允许脚本、外链和事件处理器。
- README 展示图统一位于 `docs/images/`，宽度不超过 960px、单张不超过 300 KB，不直接复用高分辨率运行资源。
- 动效只作用于小面积元素的 `transform` 和 `opacity`，并提供 `prefers-reduced-motion` 降级。

## 新增主题完成清单

- [ ] 清单、ID、颜色和所有资源均为本地可移植路径。
- [ ] 首页是完整工作区场景，不是带圆角和阴影的照片框。
- [ ] 原生项目选择器、输入操作、菜单和快捷键没有被重建或遮挡。
- [ ] 普通与窄窗口中人物、乐器和关键构图不会被二次裁切。
- [ ] 当前任务、文件变更、多行输入、运行/停止和输出面板已检查。
- [ ] 产品菜单、账号/用量菜单、消息操作菜单和对话框文字可读。
- [ ] 主题切换持久化、键盘操作和失败回滚已验证。
- [ ] 图片已压缩，展示图已生成，所有验证通过。

更底层的字段和安全约束以 `skills/codex-theme-builder/references/theme-contract.md`、`new-theme-blueprint.md`、`runtime-architecture.md` 与 `qa-checklist.md` 为准。
